# luck_rank 插件文档

`/luck` 是本插件统一前缀。

大多数命令支持有空格和无空格两种写法，例如：

- `/luck 使用绝对零度@某人`
- `/luck 使用 绝对零度 @某人`

## 管理员命令

当前代码逻辑中，仅 `admin_settings.extra_admin_qqs`（或根节点兼容字段 `extra_admin_qqs`）里配置的 QQ 可以使用管理员命令。

管理员目标统一支持以下三类：

- `自己`
- `@某人`
- `全体成员`（兼容 `全体`、`全员`）
- `@全体成员`

其中：
- `全体成员 / 全体 / 全员` 适合不想真的提醒全群、只想静默给全体已有档案成员统一发放资源或卡牌的场景。
- `@全体成员` 适合需要明确告知全群“管理员正在统一发放 / 统一授予”的场景；现在会和 `全体成员` 一样真正触发实际效果，不再只是播报。

### 管理员命令表

| 类型 | 命令格式 | 作用 | 备注 |
| --- | --- | --- | --- |
| 增加资源 | `/luck 增加 自己 命运牌 数量` | 给自己增加命运牌换牌次数。 | `数量` 支持正负数。 |
| 增加资源 | `/luck 增加 @某人 功能牌 数量` | 给目标增加功能牌免费抽取次数。 | `数量` 支持正负数。 |
| 增加资源 | `/luck 增加 全体成员 金币 数量` | 给全体已有档案成员增加或扣除金币。 | `金币 / 气运 / 运势` 都可作为金币别名。 |
| 授予功能牌 | `/luck 授予功能牌 自己 功能牌名` | 直接给自己授予指定功能牌。 | 走“额外免费抽取并抽中该牌”的播报逻辑。 |
| 授予功能牌 | `/luck 授予功能牌 @某人 功能牌名` | 直接给目标授予指定功能牌。 | 目标会获得正常抽中播报。 |
| 授予功能牌 | `/luck 授予功能牌 全体成员 功能牌名` | 给全体已有档案成员统一授予指定功能牌。 | 会发送汇总播报。 |
| 丢弃功能牌 | `/luck 丢弃功能牌 自己 功能牌名` | 强制让自己丢弃一张指定功能牌。 | 若该牌已启用，会一并清理对应状态。 |
| 丢弃功能牌 | `/luck 丢弃功能牌 @某人 功能牌名` | 强制让目标丢弃一张指定功能牌。 | 适合回收测试牌、问题牌。 |
| 丢弃功能牌 | `/luck 丢弃功能牌 全体成员 功能牌名` | 让全体已有档案成员统一丢弃指定功能牌。 | 会发送汇总播报。 |
| 授予称号 | `/luck 授予称号 自己 称号名` | 直接给自己授予指定称号。 | 授予后如需生效，继续使用佩戴命令。 |
| 授予称号 | `/luck 授予称号 @某人 称号名` | 直接给目标授予指定称号。 | 适合测试新称号效果。 |
| 授予称号 | `/luck 授予称号 全体成员 称号名` | 给全体已有档案成员统一授予指定称号。 | 会发送汇总播报。 |
| 丢弃称号 | `/luck 丢弃称号 自己 称号名` | 强制让自己丢弃一个指定称号。 | 会同时移除拥有、佩戴与手动授予记录。 |
| 丢弃称号 | `/luck 丢弃称号 @某人 称号名` | 强制让目标丢弃一个指定称号。 | 适合清理失效称号、测试称号。 |
| 丢弃称号 | `/luck 丢弃称号 全体成员 称号名` | 让全体已有档案成员统一丢弃指定称号。 | 会发送汇总播报。 |

## AOE 词条风险提示

- `aoe_damage / aoe_heal / aoe_cleanse` 这类大范围词条会同时产生大量结算、播报与战报。
- 当波及人数来到 `40+` 时，建议谨慎使用；如果配置到 `100+`，很容易造成消息刷屏、结算变慢，甚至出现功能牌看起来“卡住”的情况。
- WebUI 的功能词条库与功能牌编辑器里，都会对这类词条给出红字风险提示。

## 授予功能牌的特殊规则

- 管理员授予功能牌会被视为“额外免费抽取一次，并且抽中了指定牌”。
- 会计入该用户的功能牌抽取统计，也会触发正常的称号判定与抽中播报。
- 这类牌会以“天赐牌”形式进入库存，可以正常使用、启用、停用、丢弃。
- 天赐牌不占常规战术卡槽，但会并入战术卡槽区块显示，便于统一查看。

## 授予称号的特殊规则

- 管理员手动授予的称号会直接写入玩家拥有列表。
- 这类称号不会因为当前未满足条件而被下一次自动同步立刻回收。
- 如果要让称号效果实际生效，玩家仍需使用 `/luck 佩戴称号 称号名`。

## 管理员丢弃的特殊规则

- 管理员丢弃功能牌时，会直接从目标库存移除对应卡牌；如果该牌正处于启用状态，也会同步清理对应的防御或反制状态。
- 管理员丢弃称号时，会同时清理玩家的 `titles`、`equipped_titles` 与 `manual_titles`，避免面板残留旧称号。
- 当前端已经删除某个称号配置后，玩家下次打开 `/luck 面板` 时会自动重新同步称号；若该称号已不存在或已失效，会被自动移除。

## 关键运行数据

真实运行数据统一保存在：

```text
data/plugin_data/luck_rank/
├─ luck_data.json
├─ cards_config.json
├─ func_cards.json
├─ titles_config.json
├─ sign_in_texts.json
├─ webui_runtime_config.json
├─ cards/
└─ func_cards/
```

其中：

- `luck_data.json`：玩家档案、库存、状态、称号、战报等。
- `cards_config.json`：命运牌池配置。
- `func_cards.json`：功能牌池配置。
- `titles_config.json`：称号配置。
- `sign_in_texts.json`：签到文案与运势区间。
- `webui_runtime_config.json`：WebUI 运行配置。
