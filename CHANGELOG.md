# Changelog

## 5.4.1-Pro - 2026-05-02

本次更新以稳定性修复和命令体验优化为主，重点补齐了签到、命令前缀、管理员权限和多群决斗的几个实际问题。

### 更新公告

- 命令前缀解析已放宽，不再强制要求 `/luck` 后必须带空格。
- 现在 `/luck签到`、`/luck 签到`、`luck签到`、`luck 签到` 都可以正常触发签到。
- `global_settings.prefix` 自定义前缀已真正接入命令网关，不再只有配置没有效果。
- 签到实际回包已补齐“末尾名言”逻辑，群内真实签到与 WebUI 预览保持一致。
- 一言逻辑支持优先使用自定义词库，网络获取失败时会自动优雅降级，不再直接缺失。
- `admin_settings.use_native_admin` 已接入兼容式原生管理员判断，不再只认 `extra_admin_qqs`。
- 决斗等待状态已改为按群隔离，多群同时发起决斗不会再互相覆盖。
- WebUI 新建空白方案或内置模板方案时，已修复 `desc` 未初始化导致的后端报错。
- 功能牌、启用/停用、公开决斗、追加投入等分支已统一使用网关透传的命令串，减少硬编码 `/luck` 带来的解析偏差。

### 关联 Issue

- #1 多群同时发起决斗时全局 PENDING_DUEL 会互相覆盖
- #2 `admin_settings.use_native_admin` 配置项没有实际生效
- #3 新建空白或内置模板方案时 WebUI 后端会抛出 `UnboundLocalError`
- #4 `global_settings.prefix` 自定义前缀配置未被命令网关使用
