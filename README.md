# luck_rank 插件文档


## 插件简介

本插件是一个面向 AstrBot 群聊场景打造的一站式签到、抽卡、互动博弈插件。它不只是“做一个每日签到”，而是把每日运势、命运牌、功能牌、称号、金币经济、排行、公开决斗和协作编辑整合进同一套系统里。

你可以把它做成轻松娱乐型的群聊小游戏，也可以把它打磨成高度贴合世界观和角色设定的长期玩法系统。无论是文案、数值、卡池、称号、播报风格，还是管理流程与协作方式，这个插件都尽量把“可定制”做深、把“可用性”做好。

## 这个插件能干什么

- 提供每日签到、运势结算、金币成长和群内排行。
- 提供命运牌系统，支持每日抽取、换牌、图片绑定和牌面配置。
- 提供功能牌系统，支持攻击、防御、治疗、状态干涉、骰子玩法和公开决斗。
- 提供称号系统，支持条件、效果、佩戴数量和展示风格配置。
- 提供个人面板、金币榜、善恶榜、统计面板等完整的群内反馈。
- 提供管理员发放、回收、授予、清理等一整套运营命令。
- 提供访客协作、临时密钥、待审核草稿和 Cloudflare 临时通道，方便多人一起维护内容。

## 核心特色

### 1. 高度自定义，能非常还原你的人设与世界观

这套插件最大的特点是“几乎整套玩法都能按你的设定来改”。

- 签到文案、运势区间、宜忌事件都能自定义。
- 命运牌、功能牌、称号都能自己配置、自己命名、自己设计描述。
- 功能牌稀有度、爆率、保底、抽卡消耗、每日次数、决斗门槛都能按你的玩法调。
- 面板标题、榜单名称、分区文案、系统播报风格都可以和你的人设体系保持统一。

如果你想做一个“很像你自己作品”的插件，而不是千篇一律的通用签到插件，这一套会非常适合。
当然，我们提供了懒狗模式，可以一键导入文案和美少女图片，非常快捷方便

### 2. 用心打磨的 WebUI，真正适合长期维护

插件内置了独立 WebUI 管理界面，我真的做了很久，我真的觉得这个效果非常炫酷，求你们了，来看看来看看来看看来看看

- 总览与方案、运行配置、签到配置、命运牌档案、功能牌档案、统计、访客协作都集中在同一套界面中。
- 常用配置可以直接可视化调整，不需要反复手改 JSON。
- 支持图片上传、卡牌编辑、批量处理、未完成置顶、统计查看与预览联动。
- WebUI 独立进程运行，和主插件逻辑隔离，适合长期使用和持续迭代。

### 3. 写入轻松便捷，尽量把复杂配置做成人能顺手用的样子

简单来说就是和配词条一样，不用写什么代码，里面内置的一些功能我都给你弄好了，如果你想要更多更有趣的功能的话，可以来找到我，或者给我交一个pr，期待功能词库越来越完善

- 签到配置支持事件池、区间和预览联动，边改边看结果。
- 命运牌和功能牌都支持图片绑定、批量导入、草稿整理和待补完检查。

### 4. 协作友好，兼顾开放编辑与安全审核

如果你的卡池、美术、文案不是一个人做，这套协作设计会很实用。

- 可以创建访客身份，分配不同权限。
- 可以生成临时密钥，把 WebUI 的协作入口发给别人，而不是直接给管理密码。
- 需要审核的改动不会直接写入正式配置，而是先进入待审核区。
- 支持 Cloudflare Quick Tunnel，方便临时对外协作，同时尽量避免直接暴露服务器真实 IP。

就是一旦在群里用了这套插件的话，你的群友会非常希望给他们做一张专属的卡牌，去被别人抽到。然后找的人多的话，我就会很烦，然后就弄了个这个东西，让他们自己去干！哈！

## 快速开始

### 1. 先把 WebUI 打开，并设置端口和访问口令

只要打开了webul，超级好上手，简单又轻松

### 2. 放行端口

如果你在本地运行，通常直接访问 `127.0.0.1:端口` 即可。  
如果你在服务器运行，请确保系统防火墙、安全组、面板放行规则等都已经放通对应端口。

特别提醒 Docker 服务器用户：

- 不仅宿主机要映射端口，
- 容器内部也要确保对应端口可访问，
- 插件监听端口、Docker 端口映射、服务器放行规则这几层都要打通。

否则 WebUI 虽然启动了，外部依然访问不到（我最开始就这样，，，，，，，）

### 3. 进入 WebUI，完成基础配置

端口和密码配置好后，进入 WebUI，按页面逐项完成：

- 运行配置
- 签到配置
- 命运牌档案
- 功能牌档案
- 统计与访客协作相关设置

配置完成后，群内就可以直接通过 `/luck` 系列命令开始使用。



大多数命令支持有空格和无空格两种写法，例如：

- `/luck 使用绝对零度@某人`
- `/luck 使用 绝对零度 @某人`
在群里发个/luck 菜单 就一目了然了

## 管理员命令

当前代码逻辑中，仅 `admin_settings.extra_admin_qqs`（或根节点兼容字段 `extra_admin_qqs`）里配置的 QQ 可以使用管理员命令。

管理员目标统一支持以下三类：

- `自己`
- `@某人`
- `全体成员`（兼容 `全体`、`全员`）
- `@全体成员`

其中：
- `全体成员 / 全体 / 全员` 适合不想真的提醒全群、只想静默给全体已有档案成员统一发放资源或卡牌的场景。
- `@全体成员` 适合需要明确告知全群“管理员正在统一发放 / 统一授予”的场景；现在会和 `全体成员` 一样真正触发实际效果，不再只是播报。

## AOE 词条风险提示

- `aoe_damage / aoe_heal / aoe_cleanse` 这类大范围词条会同时产生大量结算、播报与战报。
- 当波及人数来到 `40+` 时，建议谨慎使用；如果配置到 `100+`，很容易造成消息刷屏、结算变慢，甚至出现功能牌看起来“卡住”的情况。
- WebUI 的功能词条库与功能牌编辑器里，都会对这类词条给出红字风险提示。

## 授予功能牌的特殊规则

- 管理员授予功能牌会被视为“额外免费抽取一次，并且抽中了指定牌”。
- 会计入该用户的功能牌抽取统计，也会触发正常的称号判定与抽中播报。
- 这类牌会以“天赐牌”形式进入库存，可以正常使用、启用、停用、丢弃。
- 天赐牌不占常规战术卡槽，但会并入战术卡槽区块显示，便于统一查看。

## 功能牌查询说明

- `/luck 查询 功能牌名` 会返回：稀有度、类型、描述、效果。
- 其中“描述”继续使用你在功能牌配置里手写的 `description`。
- 其中“效果”不需要手填第二份文案，会根据功能牌 `tags` 自动翻译成人话并展示给玩家。

## 授予称号的特殊规则

- 管理员手动授予的称号会直接写入玩家拥有列表。
- 这类称号不会因为当前未满足条件而被下一次自动同步立刻回收。
- 如果要让称号效果实际生效，玩家仍需使用 `/luck 佩戴称号 称号名`。

## 管理员丢弃的特殊规则

- 管理员丢弃功能牌时，会直接从目标库存移除对应卡牌；如果该牌正处于启用状态，也会同步清理对应的防御或反制状态。
- 管理员丢弃称号时，会同时清理玩家的 `titles`、`equipped_titles` 与 `manual_titles`，避免面板残留旧称号。
- 当前端已经删除某个称号配置后，玩家下次打开 `/luck 面板` 时会自动重新同步称号；若该称号已不存在或已失效，会被自动移除。

## Cloudflare 开箱即用模式补充

访客协作用 Cloudflare Quick Tunnel 隐藏 WebUI 所在机器的真实 IP。它需要一个 Cloudflare 官方小程序 `cloudflared`：

- `cloudflared` 运行时会主动连接 Cloudflare，并生成 `https://xxxx.trycloudflare.com` 临时地址。
- 协作者访问的是这个临时地址，不需要知道服务器真实 IP。
- 不要把 WebUI 管理密码发给协作者，只发访客密钥。

访客协作页现在按傻瓜流程引导：

1. 检测当前运行环境，显示系统、架构、推荐下载文件和检测结果。
2. 如果已检测到 `cloudflared`，直接点击“生成临时地址”。
3. 如果未检测到，页面给出当前环境对应的 Cloudflare 官方下载直链。
4. 用户自行下载约 30-40MB 的二进制文件，放到页面提示的插件 `bin` 目录。
5. 点击“整理文件并检测”，插件会自动复制/改名、修复执行权限并验证 `cloudflared --version`。

手动下载说明：

- 打开页面给出的官方推荐下载地址。
- Windows 通常下载 `cloudflared-windows-amd64.exe`，放进 `bin` 后插件会自动整理成 `cloudflared.exe`。
- Linux 通常下载 `cloudflared-linux-amd64`，放进 `bin` 后插件会自动整理成 `cloudflared` 并执行 `chmod +x`。
- macOS 的 `.tgz` 可以直接放到页面提示的目录，插件会自动解压并整理。
- 文件放好后点“整理文件并检测”。插件会执行 `cloudflared --version` 验证可用性。
- 也可以用 `winget`、`brew`、Linux 包管理器安装到系统 PATH，再回到页面重新检测。

身份权限可以直接在“访客协作”页新增和编辑。是否需要审核、是否允许懒狗批量生成、是否允许批量图片、单次卡片/图片上限都会随身份模板保存，生成临时密钥时直接套用该身份。

## Visitor Collaboration / 临时授权协作

WebUI 顶部新增“访客协作”入口。管理员可以预设权限身份并生成临时密钥，访客用密钥登录同一个 WebUI 后，只能按对应身份权限操作。需要审核的身份不会直接写入正式配置，所有改动会进入独立待审核区，管理员通过后才会写入真实配置文件。

默认身份包括：

- 只读访客：只能查看。
- 功能牌投稿员：可新增少量功能牌和上传少量图片，需要审核。
- 功能牌协作者：可新增和修改功能牌，需要审核。
- 受信协作者：可新增、修改、上传图片，可免审核，但不能删除核心配置。

管理员 QQ 私聊命令：

```text
/luck 管理验证 WebUI管理密码
/luck 访客身份
/luck 生成临时密钥 功能牌投稿员
/luck 待审核
/luck 同意草稿 draft_xxx
/luck 拒绝草稿 draft_xxx
```

这些命令必须在私聊中使用，并且发送者必须是插件管理员。生成密钥和审核草稿前，需要先用 WebUI 管理密码完成二次验证。

### Cloudflare Quick Tunnel

其它兜底方式：

- Windows 可尝试：`winget install --id Cloudflare.cloudflared`
- macOS 可尝试：`brew install cloudflared`
