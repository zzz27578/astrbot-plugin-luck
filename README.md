# luck_rank 插件文档

`/luck` 是本插件统一前缀。

大多数命令支持有空格和无空格两种写法，例如：

- `/luck 使用绝对零度@某人`
- `/luck 使用 绝对零度 @某人`

## 管理员命令

当前代码逻辑中，仅 `admin_settings.extra_admin_qqs`（或根节点兼容字段 `extra_admin_qqs`）里配置的 QQ 可以使用管理员命令。

管理员目标统一支持以下三类：

- `自己`
- `@某人`
- `全体成员`（兼容 `全体`、`全员`）
- `@全体成员`

其中：
- `全体成员 / 全体 / 全员` 适合不想真的提醒全群、只想静默给全体已有档案成员统一发放资源或卡牌的场景。
- `@全体成员` 适合需要明确告知全群“管理员正在统一发放 / 统一授予”的场景；现在会和 `全体成员` 一样真正触发实际效果，不再只是播报。

### 管理员命令表

| 类型 | 命令格式 | 作用 | 备注 |
| --- | --- | --- | --- |
| 增加资源 | `/luck 增加 自己 命运牌 数量` | 给自己增加命运牌换牌次数。 | `数量` 支持正负数。 |
| 增加资源 | `/luck 增加 @某人 功能牌 数量` | 给目标增加功能牌免费抽取次数。 | `数量` 支持正负数。 |
| 增加资源 | `/luck 增加 全体成员 金币 数量` | 给全体已有档案成员增加或扣除金币。 | `金币 / 气运 / 运势` 都可作为金币别名。 |
| 授予功能牌 | `/luck 授予功能牌 自己 功能牌名` | 直接给自己授予指定功能牌。 | 走“额外免费抽取并抽中该牌”的播报逻辑。 |

## AOE 词条风险提示

- `aoe_damage / aoe_heal / aoe_cleanse` 这类大范围词条会同时产生大量结算、播报与战报。
- 当波及人数来到 `40+` 时，建议谨慎使用；如果配置到 `100+`，很容易造成消息刷屏、结算变慢，甚至出现功能牌看起来“卡住”的情况。
- WebUI 的功能词条库与功能牌编辑器里，都会对这类词条给出红字风险提示。

## 授予功能牌的特殊规则

- 管理员授予功能牌会被视为“额外免费抽取一次，并且抽中了指定牌”。
- 会计入该用户的功能牌抽取统计，也会触发正常的称号判定与抽中播报。
- 这类牌会以“天赐牌”形式进入库存，可以正常使用、启用、停用、丢弃。
- 天赐牌不占常规战术卡槽，但会并入战术卡槽区块显示，便于统一查看。

## 功能牌查询说明

- `/luck 查询 功能牌名` 会返回：稀有度、类型、描述、效果。
- 其中“描述”继续使用你在功能牌配置里手写的 `description`。
- 其中“效果”不需要手填第二份文案，会根据功能牌 `tags` 自动翻译成人话并展示给玩家。

## 授予称号的特殊规则

- 管理员手动授予的称号会直接写入玩家拥有列表。
- 这类称号不会因为当前未满足条件而被下一次自动同步立刻回收。
- 如果要让称号效果实际生效，玩家仍需使用 `/luck 佩戴称号 称号名`。

## 管理员丢弃的特殊规则

- 管理员丢弃功能牌时，会直接从目标库存移除对应卡牌；如果该牌正处于启用状态，也会同步清理对应的防御或反制状态。
- 管理员丢弃称号时，会同时清理玩家的 `titles`、`equipped_titles` 与 `manual_titles`，避免面板残留旧称号。
- 当前端已经删除某个称号配置后，玩家下次打开 `/luck 面板` 时会自动重新同步称号；若该称号已不存在或已失效，会被自动移除。

## 关键运行数据

真实运行数据统一保存在：

```text
data/plugin_data/luck_rank/
├─ luck_data.json
├─ cards_config.json
├─ func_cards.json
├─ titles_config.json
├─ sign_in_texts.json
├─ webui_runtime_config.json
├─ cards/
└─ func_cards/
```

其中：

- `luck_data.json`：玩家档案、库存、状态、称号、战报等。
- `cards_config.json`：命运牌池配置。
- `func_cards.json`：功能牌池配置。
- `titles_config.json`：称号配置。
- `sign_in_texts.json`：签到文案与运势区间。
- `webui_runtime_config.json`：WebUI 运行配置。

## Cloudflare 开箱即用模式补充

访客协作用 Cloudflare Quick Tunnel 隐藏 WebUI 所在机器的真实 IP。它需要一个 Cloudflare 官方小程序 `cloudflared`：

- `cloudflared` 运行时会主动连接 Cloudflare，并生成 `https://xxxx.trycloudflare.com` 临时地址。
- 协作者访问的是这个临时地址，不需要知道服务器真实 IP。
- 不要把 WebUI 管理密码发给协作者，只发访客密钥。

访客协作页现在按傻瓜流程引导：

1. 检测当前运行环境，显示系统、架构、推荐下载文件和检测结果。
2. 如果已检测到 `cloudflared`，直接点击“生成临时地址”。
3. 如果未检测到，可以选择“自动下载”或“手动下载”。
4. 自动下载会把文件放到插件数据目录的 `bin` 文件夹，验证 `cloudflared --version` 后记录路径。
5. 手动下载会展示当前环境对应的官方下载地址、可接受文件名、应该放置的目录和推荐放置路径。把文件放好后点击“重新检测”即可。

自动下载说明：

- 默认使用 Cloudflare 官方 GitHub Release 下载适配当前系统架构的二进制文件。
- 下载失败时，页面会保留安装日志和失败原因，并提供继续/重新下载、清理后重新安装。
- 取消安装会尝试清理本次下载留下的残留文件。
- 如果运行环境连不上 GitHub，可以在 WebUI 里把下载源切到 `custom`，填写镜像模板。模板支持 `{url}` 和 `{asset}`，例如 `https://你的镜像/{url}` 或 `https://你的镜像/latest/download/{asset}`。
- 点击自动下载时会先保存页面上的下载源设置，并使用页面显示的“当前实际下载地址”。
- 自动安装不会注册系统服务，不会修改系统 PATH，不会改防火墙；下载完成后插件会记录该二进制路径并用于启动临时通道。

手动下载说明：

- 打开页面给出的官方推荐下载地址。
- Windows 通常下载 `cloudflared-windows-amd64.exe`，也可以改名为 `cloudflared.exe`。
- Linux 通常下载 `cloudflared-linux-amd64`，也可以改名为 `cloudflared`。
- macOS 的 `.tgz` 需要先解压，把里面名为 `cloudflared` 的文件放到页面提示的目录。
- 文件放好后点“我已放好，重新检测”。插件会执行 `cloudflared --version` 验证可用性。
- 也可以用 `winget`、`brew`、Linux 包管理器安装到系统 PATH，再回到页面重新检测。

身份权限可以直接在“访客协作”页新增和编辑。是否需要审核、是否允许懒狗批量生成、是否允许批量图片、单次卡片/图片上限都会随身份模板保存，生成临时密钥时直接套用该身份。

## Visitor Collaboration / 临时授权协作

WebUI 顶部新增“访客协作”入口。管理员可以预设权限身份并生成临时密钥，访客用密钥登录同一个 WebUI 后，只能按对应身份权限操作。需要审核的身份不会直接写入正式配置，所有改动会进入独立待审核区，管理员通过后才会写入真实配置文件。

默认身份包括：

- 只读访客：只能查看。
- 功能牌投稿员：可新增少量功能牌和上传少量图片，需要审核。
- 功能牌协作者：可新增和修改功能牌，需要审核。
- 受信协作者：可新增、修改、上传图片，可免审核，但不能删除核心配置。

管理员 QQ 私聊命令：

```text
/luck 管理验证 WebUI管理密码
/luck 访客身份
/luck 生成临时密钥 功能牌投稿员
/luck 待审核
/luck 同意草稿 draft_xxx
/luck 拒绝草稿 draft_xxx
```

这些命令必须在私聊中使用，并且发送者必须是插件管理员。生成密钥和审核草稿前，需要先用 WebUI 管理密码完成二次验证。

### Cloudflare Quick Tunnel

如果不想暴露服务器 IP，可以使用 Cloudflare Quick Tunnel。WebUI 的“访客协作”页面会检测 `cloudflared` 是否存在，并给出推荐命令：

```bash
cloudflared tunnel --url http://127.0.0.1:4399
```

如果 WebUI 端口不是 `4399`，请替换为实际端口。命令运行后会生成 `trycloudflare.com` 临时地址，把它填回“访客协作”页面即可。

其它兜底方式：

- Windows 可尝试：`winget install --id Cloudflare.cloudflared`
- macOS 可尝试：`brew install cloudflared`
- Linux 请按 Cloudflare 官方文档使用包管理器或下载对应架构二进制。
- 也可以从页面给出的官方地址手动下载，把文件放到页面提示的插件 `bin` 目录后重新检测。
- 也可以填写 ngrok、frp、反向代理、自有公网地址。
- 不需要公网时，也可以只使用本地访客权限和审核系统。

注意：Quick Tunnel 地址只在 `cloudflared` 运行期间有效，重启后可能变化。不要把 WebUI 管理密码发给访客，普通协作建议使用“需要审核”的身份。
