# luck_rank 插件文档（用户视角 + 管理员指令）

> 前缀统一为：`/luck`
> 
> 大部分指令 **支持有无空格**：
> - ` /luck 使用绝对零度@某人 `
> - ` /luck 使用 绝对零度 @某人 `
> 两种都可用。

---

## 一、普通用户指令（完整）

### 1) 基础金币玩法

- `/luck 运势`
  - 每日签到，获取运势金币。
- `/luck 抽卡`
  - 命运牌抽卡，直接加/减金币（每日次数由配置决定）。
- `/luck 金币榜`
  - 查看金币排行榜。
- `/luck 菜单`
  - 查看简明指令菜单。
- `/luck 帮助`
  - 查看详细规则说明。

### 2) 功能牌玩法

- `/luck 抽取功能牌`
  - 抽取功能牌（每日免费次数 + 额外付费次数受配置影响）。
- `/luck 面板`
  - 查看个人面板（金币、称号、状态、骰局状态、卡槽、战报）。
- `/luck 善恶榜`
  - 查看善恶排行。

- `/luck 使用卡名@目标`
  - 使用攻击/治疗牌。
  - 例：`/luck 使用绝对零度@张三`
- `/luck 使用卡名随机`
  - 攻击牌可盲打。
  - 例：`/luck 使用掠夺之手随机`

- `/luck 启用卡名`
  - 启用防御牌。
- `/luck 停用卡名`
  - 停用防御牌。
- `/luck 丢弃卡名`
  - 丢弃卡牌，腾卡槽。

### 3) 骰子与对赌

- `/luck 对赌@某人 金额`
  - 发起公开对赌，默认每日限次。
  - 赌注必须落在配置的上下限范围内。
  - 免费公开局必须双方都明确确认，不会自动代打。

- `/luck 确认`
  - 接受当前赌注；若对方已加注，则表示接下加注继续开局。

- `/luck 加注 金额`
  - 仅被挑战者可在应战阶段加注。
  - 新赌注不能低于当前赌注，且不能超过配置上限。

## 二、管理员/开发者指令（完整）

> 当前代码逻辑：仅 `extra_admin_qqs` 中配置的 QQ 可用（逗号分隔）。

### `/luck 增加 @某人 类型 数值`

支持类型：`命运牌`、`功能牌`、`金币`（或 `气运`、`运势`）

示例：

- `/luck 增加 @某人 命运牌 3`
  - 给目标增加命运牌换牌机会。
- `/luck 增加 @某人 功能牌 1`
  - 给目标增加功能牌免费抽取次数。
- `/luck 增加 @某人 金币 50`
  - 给目标加金币。
- `/luck 增加 @某人 金币 -50`
  - 扣目标金币。

## 三、关键配置说明

### 1) 配置文件分布

- `config/cards_config.json`：命运牌配置
- `config/func_cards.json`：功能牌配置（含标签 tags）
- `config/dice_rules.json`：骰子规则配置（点数区间、结果、payload）

### 2) 关键开关（`func_cards_settings`）

- `enable`
  - 功能牌系统总开关。
- `enable_dice_cards`
  - 骰子功能牌开关（关闭后过滤骰子牌并禁止使用）。
- `enable_public_duel_mode`
  - 公开对赌开关（无需功能牌的日常骰局）。
- `public_duel_daily_limit`
  - 公开对赌每日发起上限。
- `public_duel_min_stake`
  - 公开对赌最低赌注。
- `public_duel_max_stake`
  - 公开对赌最高赌注。
- `economy_settings.draw_probability`
  - 功能牌基础掉率。
- `economy_settings.free_daily_draw`
  - 每日免费抽功能牌次数。
- `economy_settings.draw_cost`
  - 额外付费抽取消耗。
- `economy_settings.pity_threshold`
  - 保底阈值。

## 四、对赌机制说明（当前版本）

- 同时只允许 1 场公开对局。
- 公开对赌支持自由下注，但必须落在配置的上下限之间。
- 发起后被挑战者可直接 `/luck 确认`，也可 `/luck 加注 金额` 抬价。
- 若被挑战者加注，则发起者必须再次 `/luck 确认` 才会正式开局。
- 日常公开局不会自动迎战，超时即流局。
- 正式开局后会分阶段播报：先开桌、再依次掷骰、最后结算。

## 六、隐私与开源发布注意事项（重要）

你准备上传 GitHub 时，建议：

- 不要提交 `data/luck_data.json`（包含用户ID、昵称、资产、战报）
- 不要提交你私有卡面资源（`assets/cards/*`、`assets/func_cards/*`）
- 不要提交你私有命运牌池配置（`config/cards_config.json`，如果里面含私货）

本仓库已提供 `.gitignore`，默认会忽略以上私有内容，并保留 `.gitkeep` 目录占位。

## 七、用户数据结构与兼容性说明

- 用户数据主文件：`data/luck_data.json`
- 当前版本在原有字段基础上增加了新字段（如状态、卡槽、善恶值、对赌计数等）。
- `LuckBank` 内置了迁移与兜底补齐逻辑：
  - 老数据可自动补字段；
  - 旧字段 `total_score / last_drawn_value` 会迁移到新字段；
  - 战报自动保留最近 3 天。

结论：
- **数据结构确实扩展了**，但做了向后兼容，老档可平滑升级。
